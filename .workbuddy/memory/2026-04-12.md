# 2026-04-12 部署问题总结

## 今日部署：本地 → 阿里云服务器

### 问题清单与解决方案

#### 1. GitHub 连接失败
- **现象**：无法从 GitHub 拉取代码
- **解决**：使用本地已有的 docker-compose.yml 手动修改配置

#### 2. MySQL 权限问题
- **现象**：Access denied for user
- **解决**：直接使用 root 用户，避免创建新用户的权限麻烦

#### 3. MySQL 8.0 语法不兼容
- **现象**：`GRANT ... IDENTIFIED BY` 语法报错
- **解决**：改用 `ALTER USER ... IDENTIFIED BY`

#### 4. 字符集乱码问题
- **现象**：character_set_client/connection/results 为 latin1
- **解决**：docker-compose.yml 添加 `characterEncoding=utf-8` 参数

#### 5. SQL 文件 BOM 头问题
- **现象**：`ASCII '\0' appeared` 错误
- **解决**：`sed -i '1s/^\xEF\xBB\xBF//'` 去掉 BOM 头

#### 6. 菜单数据不完整
- **现象**：服务器 24 条菜单 vs 本地 43 条
- **解决**：创建 menu_full.sql 补全数据，包含完整菜单和角色权限

#### 7. 前端代码不同步
- **现象**：上传了文件但页面没更新
- **原因**：.user.ini 文件保护 + 缓存问题
- **解决**：`chattr -i .user.ini` 移除保护，重新构建

#### 8. 前端构建失败 - API 导出缺失
- **现象**：`getList is not exported from '@/api/storage'`
- **原因**：storage.js 和 collection.js 缺少 getList 导出
- **解决**：添加 getList 作为 getStorageList/getCollectionList 的别名

#### 9. 路径确认问题
- **现象**：用户纠正文件路径描述错误
- **教训**：storage/info/index.vue ≠ storage/index.vue，描述要准确
